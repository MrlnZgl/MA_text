\paragraph{Mesh Generation}
The transfer of the curved TBL simulation setup to \name{OpenFOAM} includes some adaptations which will be presented here. 
First, the mesh has to be regenerated. 
It is not possible to use a mesh created in \name{gmsh} and transfer it to \name{OpenFOAM}, since the code has to adapt the domain and the mesh after every iteration of the optimization.
Therefore, the domain has to be regenerated in every iteration with \verb|blockMesh|.
As a consequence the domain has to be defined in \verb|blockMeshDict|.
The plane part is meshed with rectangular blocks.
The domain close to the lower wall has a progression factor included such that the resolution increases towards the wall. 
In the curved part the lower wall is defined through a circle arc similar to the \name{gmsh} mesh used in \name{neko}. 
The upper wall is determined by a spline. 
The points defining the spline are the coordinates $x_{\theta_i}$, $y_{\theta_i}$ of the \benennung{design parameters} $\theta_i$ whose position is modified to minimize the \benennung{objective function}.  
The optimization code is constructed such that it modifies position of the \benennung{design parameters} in wall-normal direction. 
Since the \benennung{design space} is curved, the wall-normal direction does not correspond to a fixed axis. 
Hence, their position has to be transformed in $x-$ and $y-$ coordinates to define the spline correctly in space.
Compared to the domain used for the DNS, the height of the domain is increased to $\delta_{99}*XX = XX$.
This modification brings some simplifications when the results of the optimized \name{OpenFOAM} simulation are transferred to the DNS domain which will be described later (see XX).

\paragraph{Initial conditions or boundary condition ??XX}
The quantities that need predefined initial conditions depend on the chosen RANS model (XXQuelle). 
Since in this project the $k-\omega-sst$-model is used, it is necessary to define initial conditions for turbulent kinetic energy $k$, dissipation $\omega$ and velocity $U$. 
This is done due to the imposition of an initial field at the inflow of the domain which is extracted from the previous DNS simulation of the curved TBL (see XX).
In the postprocessing of the DNS simulaiton the streamwise position in the plane part of the domain where $Re_{theta} = 1000$ is reached is identified. 
At this streamwise position the fields of $U$, $k$ and $\omega$ are extracted along the height of the domain.
In order to do this the postprocessing of the DNS simulation data is repeated for the plane at $Re_{theta} = 1000$. 
With the \name{Python} package \benennung{fluidfoam} it is possible to read an \name{OpenFOAM} mesh. 
This is used to extract the points of the \name{OpenFOAM} mesh at inflow.
These points are used as interpolation points to perform the postprocessing. 
As described earlier the domain used in \name{OpenFOAM} is higher than the one for the DNS simulations.
The usage of this higher domain would lead to interpolation conflicts since there are no data for the upper interpolation points. 
To avoid these conflicts the interpolation is performed on an \name{OpenFOAM} domain which has the same height as the \name{neko} domain. 
The values for $U$ can be extracted directly, whereas $k$ and $\omega$ need to be computed. 
To use these fields in the higher \name{OpenFOAM} domain additional values for the upper part of the inflow are manually added. 
For this part of the inflow the last interpolated value is copied and added in the remaining cells.
