\section{Channel simulations}\label{sec:channel}
In the beginning of the project channel simulations were performed. 
To get familiar with the \acrshort{dns} code and the effect of different settings these reproducable simulations were performed.
In addition, the created postprocessing script was verified with these results and reference results.
In the \name{neko} installation some examples are included which also contains one for a tubulent chnannel simulation.
We used this example as starting point and made some modifications.
The simulation is setup for a turbulent channel flow at $Re_\tau = 180$.
With the bulk velocity $U_b = 1$ and the channel half height $\delta = 1$, the bulk Reynolds number is set to be $Re_b = 2800$.
The simulation runs with polynomial order 7 and time order 3.
At the top and the bottom wall we set \benennung{no-slip} as boundary condition.
The streamwise and spanwise wall are set to be periodic, meaning that everything that goes out at one side enters the domain at the opposite.
As a first step we refined the mesh which was necessary to create proper statistics.
For an adequate resolution we defined the grid spacing in plus units to be $\Delta x^+= 12$, $\Delta z^+= 6$. 
In wall-normal direction we set the point closest to the wall to be at $y^+ = 1$ and then coarsening the resolution.
The physical domain size is $(4\pi \delta, 2 \delta, \frac{4}{3}\pi \delta)$.
Then we applied some new initial conditions to create the turbulence.
We added more pertubations in different scales such that the transition into a fully turbulent flow is much faster.
In addition, the new pertubations made the old random pertubations in wall-normal direction become unnecessary (XX warum ist das gut??).
The new setup of initial conditions is depicted in \autoref{fig:channel_IC}.
With this setup simulations were performed.
The collected statistics are discussed in chapter XX.

\section{Plane turbulent boundary layer simulations}\label{sec:planeTBL}

\subsection{Simulation setup}

In the following part of the project we created a setup for a simulation of a \acrshort{tbl} over a plane wall.
These simulations are used to verify the general settings for \acrshort{tbl} simulations, and to verify the results through the comparison with similar simulations.
The domain is defined as a box of the dimension $325 \times 24.5 \times 24.5$ which corresponds to $50\delta_{99} \times 4\delta_{99} \times 4\delta_{99}$ measured at a reference point at $Re_{\theta} = 1000$.
The resolution in streamwise and spanwise direction is constant with $x^+= 12$ and $z^+ = 6$.
In wall-normal direction the resolution decreases with increasing distance from the lower wall starting at the wall with a resolution of $y^+ = 0.8$. 
In the middle the resolution is constant on a quite coarse level.
At the upper wall there is a small area where the resolution again increases since there the shear stress is measured.
The mesh is created with \name{gmsh}. 
First a 2D mesh is constructed in the $x-y$/ streamwise- wall-normal-plane is created. 
With the mesh transform tool in \name{neko} it is then extended in the spanwise direction. 

\subsubsection{Boundary and initial conditions}

To setup the \acrlong{tbl} boundary and initla conditions are necessary. 
In the initial condition a boundary layer is imposed at the lower wall. 
Since a laminar boundary layer has the quite easy blasuis profile as solution, a laminar boundary layer is given as initial velocity condition. 
The velocity in the free-stream $U_0$ is set to be 1 initially.
AS boundary conditions we impose a no-slip condition on the lower wall.
The outflow has the boundary condition 'outlfow+dong', which means XXX.
For the upper wall we tested two different setups.
For one simulation we put a 'normal outflow' condition at the top wall, which allows outflow in normal direction across the domain end.
In the otehr simulation we imposed a Dirichlet condition with (1,0,0) as velocity profile. 
This is done to compare the effect of the different boundary conditions on the flow behaviour. 
This becomes especially importnat for the simulation of the curved \acrshort{tbl}, since there the boundary condition needs to be imposed piecewise on the top wall.

\subsubsection{Case settings}

The additional settings for the simulation are definde in the CASE-file. 
The polynomial order used for the simulations is 7 and the time order is 3.
To enable shorter simulations times, the simulations first are started with lower polynomial order during the developing time of the \acrshort{tbl}.
When the \acrshort{tbl} is stable the polynomial order is increased for higher result accuracy.
All settings in the case file are listed in XXANHANG.

\autoref{fig:planeTBLverifU} depicts the velocity in inner scales.
It can be seen that all three profiles follow the analytical definitions of the linear law and the log law.
The reference profile differs from the two performed simulations in the region of the outer flow, which can be explained through the lower $Re_{\theta}$ of the reference data.
Besides from that, no differences between the profiles can be observed.
For the fluctuations similar observation holds.
In the viscous sublayer the profiles coincide for all fluctuation components.
In the wake region it can be seen that the reference values are slightly below the two performed simulation data, which again be explained with the lower $\mathrm{Re}_{\theta}$. \\
The presented results show a proper agreement of both simulations with the reference data.
Based on that two conclusions can be drawn.
First, the simulations seem to generally create reasonable results compared to other \acrshort{tbl} simulations.
And second, the difference between the two simulation with different boundary conditions at the top wall are negligible small.
